# Дизайн: Webinar Flow + презентация + правни условия

Дата: 2026-07-06 · Статус: одобрен устно („давай, действаме") и внедрен в същата сесия.

## Цел

Надграждане на уебинар фунията (виж docs/WEBINAR-FUNNEL.md) с:
1. **Webinar Flow** — date-triggered автоматизации: задаваш дата → имейлите тръгват сами.
2. **Презентация за уебинара** — 90 мин, по „Perfect Webinar" скрипта от архива на
   собственика (`D:\Работа Про Маркетинг Онлайн\Други\Уебинар Фуния\Скрипт_за_уебинар_1-*.docx`).
3. **Правни условия за checkout** — защита при продажба на курсове онлайн (ЗЗП/ЕС).

## 1. Webinar Flow

**Подход (избран):** Vercel cron (на всеки час) + стъпки с прозорци, дедуп през
`contact_activities`. Алтернативи, отхвърлени: n8n на VPS (нова инфраструктура за нещо,
което Vercel вече прави) и Supabase pg_cron (логиката е в Next.js, не в SQL).

- `lib/webinar/flow.ts` — стъпки: −3д, −1д, −1ч, +3ч (оферта 48ч), +27ч (последен шанс).
  Всяка стъпка има прозорец [open → sendUntil]: късно записалите не получават стари
  напомняния; пропуснат час на cron-а се наваксва на следващото завъртане.
- Дедуп: activity `webinar_email_{stage}` на контакт. Грешка при изпращане → без activity
  → автоматичен retry на следващия час.
- `/api/cron/webinar-flow` (GET, CRON_SECRET) + запис в vercel.json (`0 * * * *`).
- Без дата (`dateISO: null`) flow-ът е безвреден no-op.

## 2. Презентация

- `/webinar/prezentacia` (noindex) + `components/webinar/WebinarDeck.tsx`.
- 40 слайда ≈ 90 мин, структура по скрипта: рапорт (5 хода) → голямо домино →
  квалификация → лична история → кейс → 4 „тайни" (нашите 4 системи, всяка:
  история → механика → оборване на убеждение → затвърждаване) → разрешение за оферта →
  стек (5 елемента, 1 540 €) → „бихте ли платили" ×3 → специална цена 397 € →
  гаранция 14 дни → 2 варианта → спешност → Q&A.
- Всеки слайд носи `notes` (какво казваш) — клавиш N. Таймер, прогрес, F за цял екран.
- Съдържанието на офертата чете цените от `lib/webinar/config.ts`.

## 3. Правни условия

- `/usloviya-kursove` — Общи условия за онлайн курсове: идентификация на продавача,
  цени, плащане (Stripe), 14-дневно право на отказ + изключението за цифрово съдържание
  (чл. 57, т. 13 ЗЗП) с изрично съгласие при плащане, наша доброволна 14-дневна
  гаранция, интелектуална собственост, **без гаранция за резултати/доходи**, ограничение
  на отговорността, ЛД/GDPR препратка, КЗП + ЕС ODR платформа.
- Съгласието се събира на 2 места: текст под всеки CheckoutButton + `custom_text.submit`
  в Stripe Checkout сесията.
